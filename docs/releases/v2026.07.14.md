# v2026.07.14

这是 XG-040G-MD OpenWrt One-KVM 固件的首个正式版本，面向已经使用 tcboot
Web U-Boot 或 tcboot OpenWrt 的 Nokia XG-040G-MD。

## 相比 rc1 的主要变化

- 新增“系统 -> 交换性能”LuCI 页面，默认使用 OpenWrt 全核 RPS 分担
  `eth0`、`eth1` 的 RX 处理，并显示队列掩码、`NET_RX`、温度和桥成员状态。
- CPU 默认并始终以原厂 1200 MHz 启动。1300/1400 MHz 只作为需要输入
  `OVERCLOCK` 明确确认的实验功能，默认不会加载 PLL 模块。
- 修复交换性能 RPC 参数读取和状态查询，实机确认 64 个 RX 队列掩码均为 `f`。
- 修复 GitHub Actions 恢复 feeds 缓存后因容器 UID 不同产生的 Git
  `dubious ownership` 构建失败，发布产物继续固定源码和 feeds commit。

## 主要功能

- 默认把 LAN2、LAN3、LAN4 和 2.5G 口组成透明交换机，通过上游 DHCP、稳定
  `.local` 名称或 IPv4LL 管理。
- 可在 LuCI 中持久选择任意单个网口作为 PXE 专用口，也可选择“不启用 PXE”
  恢复四口交换机模式。
- 内置 One-KVM 0.2.3、中文 LuCI、MS2109 UVC/USB Audio 支持，以及
  CH340/CH341 + CH9329 键鼠控制。
- 支持 H.264、H.265、VP8 和 VP9 软件编码，并提供 ttyd、GOSTC、EasyTier、
  FRPC、GPIO、串口/HID relay 与 Wake-on-LAN 运行依赖。
- 支持 USB3 `KVMSTORE` 本地 PXE 仓库，以及通过 rclone 连接 WebDAV 和其他
  remote 的无硬盘 Cloud PXE。
- 已完成 FirPE、SystemRescue、UEFI/BIOS iPXE、真实 WebDAV、MS2109、
  CH9329、系统统计和温度监控的实机验证。
- LuCI 提供 One-KVM 状态、版本、恢复、数据重置、PXE、Cloud PXE、FRPC、
  系统统计、温度和交换性能管理界面。

## 下载选择

- `xg040g-onekvm-factory.bin`：通过 tcboot Web U-Boot 刷入完整版，会重建数据。
- `xg040g-onekvm-sysupgrade.bin`：从现有 tcboot OpenWrt 升级到完整版；通常不加
  `-n`，以保留密码和 One-KVM 数据。
- `xg040g-minimal-factory.bin` 和 `xg040g-minimal-sysupgrade.bin`：仅用于首次
  启动验证或恢复，不包含完整 One-KVM 运行环境。
- `one-kvm-*.apk`、`luci-app-one-kvm-*.apk` 和
  `luci-i18n-one-kvm-zh-cn-*.apk`：仅用于相同运行时 ABI 的完整版固件单包升级。
- `source-with-submodules.tar.zst`：包含固定 OpenWrt 与 One-KVM submodule 的
  可归档源码包。

刷写前必须使用 Release 中的 `SHA256SUMS` 校验下载文件。仅支持
`nokia_xg-040g-md-tcboot` profile，不要刷入其他型号、原厂分区或非 tcboot
设备。

## 已知限制与待测项

- AN7581 UDC/OTG endpoint 不可用，因此不支持 USB gadget 键鼠或 USB
  mass-storage 虚拟介质。键鼠使用 CH9329，安装镜像和应急系统使用 PXE。
- AN7581 没有可用的 OpenWrt 硬件视频编码后端，本版本提供软件编码。约
  320 MiB 内存不适合高分辨率软件编码，不承诺 1080p、2K 或 4K 实时性能。
- USB3 `KVMSTORE` 的 5 Gbit/s 链路、自动挂载和持续读写仍待完整硬件闭环。
- 动态 PXE 端口的配置、运行态切换和持久化已经实机验证；新版拓扑下从实体
  客户端启动 FirPE/SystemRescue 仍待复测。旧固定 LAN4 拓扑已完成完整启动链路。
- 动态 PXE 切换后的持续多流量 RPS 回归、2.5G 聚合吞吐与 24 小时压力测试
  尚未完成，因此不宣称 2.5G 软件交换达到线速。
- 1300/1400 MHz 超频没有纳入正式版验收，保持关闭。发布默认值为 1200 MHz。
